# 团队工作流程 (TEAMWORK_FLOW) — 小型研发

## 任务流水线

```
需求/拆解(项目经理) → 开发(研发) → 测试(测试) → 验收(项目经理)
```

## 阶段定义

| 阶段 | 默认执行人 | 产出要求 |
|------|-----------|---------|
| 需求/拆解 | 项目经理 | 任务描述、验收标准、功能点 |
| 开发 | 研发 | 代码提交完毕，PR 链接，自测通过 |
| 测试 | 测试 | 测试报告，Bug 列表（若有），回归完成 |
| 验收 | 项目经理 | 验收结论，上线确认 |

## 流转规则

### 正常流转

1. 任务"进展"变为"已完成"后，主 Agent 负责流转
2. 流转时：更新"任务执行人"、"任务阶段"、"进展"为待开始；可在"最新进展记录"中添加流转备注
3. 主 Agent 可根据任务复杂度合并或跳过阶段

### 回退规则

- 测试阶段发现 Bug → 打回"开发"阶段
- 验收不通过 → 根据具体情况打回相应阶段

## 主 Agent 职责

- 定期检查任务表；检查"已完成"任务 → 按流水线流转
- 检查"进行中"超时任务 → 评估是否需要介入
- 状态变化时在飞书群同步简报

## 任务优先级

- **P0**: 紧急 **P1**: 高 **P2**: 正常（默认） **P3**: 低
