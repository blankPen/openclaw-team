# 团队工作流程 (TEAMWORK_FLOW)

## 任务流水线

```
需求分析(产品) → 技术设计(可跳过) → 开发(前端/后端) → 测试(测试) → 验收(项目经理)
```

## 阶段定义

| 阶段 | 默认执行人 | 产出要求 |
|------|-----------|---------|
| 需求分析 | 产品 | PRD/需求文档，明确验收标准，功能点列表 |
| 技术设计 | 前端研发/后端研发 | 技术方案文档，接口设计，数据库设计 |
| 开发 | 前端研发/后端研发 | 代码提交完毕，PR 链接，自测通过 |
| 测试 | 测试 | 测试报告，Bug 列表（若有），回归完成 |
| 验收 | 项目经理 | 验收结论，上线确认 |

## 流转规则

### 正常流转

1. 任务"进展"变为"已完成"后，主 Agent 负责流转
2. 流转时：
   - 将"进展"重置为"待开始"
   - 更新"任务执行人"为下一阶段负责人
   - 更新"任务阶段"
   - 可在"最新进展记录"中添加流转备注
3. 主 Agent 可根据任务复杂度跳过阶段（如小需求跳过"技术设计"）

### 回退规则

- 测试阶段发现 Bug → 打回"开发"阶段
- 验收不通过 → 根据具体情况打回相应阶段

### 流转示例

```bash
# 需求分析完成，流转到开发
openclaw-team task flow recXXX --next-assignee "前端研发" --stage "开发"

# 测试完成，流转到验收
openclaw-team task flow recXXX --next-assignee "项目经理" --stage "验收"

# 测试不通过，打回开发
openclaw-team task flow recXXX --next-assignee "前端研发" --stage "开发" --note "存在 XSS 漏洞，需修复"

# 验收通过，标记完成
openclaw-team task flow recXXX --next-assignee "项目经理" --stage "完成"
```

## 主 Agent 职责

- 每 10 分钟检查一次任务表
- 检查"已完成"任务 → 按流水线流转
- 检查"进行中"超 24 小时任务 → 评估是否需要介入
- 状态变化时在飞书群同步简报

## 任务优先级

- **P0**: 紧急，需立即处理，阻塞其他任务
- **P1**: 高优先级，当日完成
- **P2**: 正常优先级（默认）
- **P3**: 低优先级，可推迟
