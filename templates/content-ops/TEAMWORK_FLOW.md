# 团队工作流程 (TEAMWORK_FLOW) — 内容运营

## 任务流水线

```
需求/选题(产品) → 文案撰写(文案) → 运营发布(运营) → 验收(项目经理)
```

## 阶段定义

| 阶段 | 默认执行人 | 产出要求 |
|------|-----------|---------|
| 需求/选题 | 产品 | 选题说明、目标受众、验收标准、渠道与节奏 |
| 文案撰写 | 文案 | 成稿文案、适用渠道、注意事项与修改说明 |
| 运营发布 | 运营 | 发布记录、数据范围、效果结论与后续建议 |
| 验收 | 项目经理 | 验收结论，闭环确认 |

## 流转规则

### 正常流转

1. 任务"进展"变为"已完成"后，主 Agent 负责流转
2. 流转时：更新"任务执行人"、"任务阶段"、"进展"为待开始；可在"最新进展记录"中添加流转备注
3. 主 Agent 可根据任务类型合并或跳过阶段（如纯文案修订可不经运营直接验收）

### 回退规则

- 运营反馈文案需修改 → 打回"文案撰写"阶段
- 验收不通过 → 根据具体情况打回相应阶段

## 主 Agent 职责

- 定期检查任务表；检查"已完成"任务 → 按流水线流转
- 检查"进行中"超时任务 → 评估是否需要介入
- 状态变化时在飞书群同步简报

## 任务优先级

- **P0**: 紧急 **P1**: 高 **P2**: 正常（默认） **P3**: 低
